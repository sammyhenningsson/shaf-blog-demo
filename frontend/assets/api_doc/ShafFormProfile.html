The ShafForm profile

This profile describes how a set of descriptors should be interpreted to turn a generic media type into a form (similar to HTML forms). For example, the HAL media type (application/hal+json) does not specify any semantics about forms, however we can add semantics about forms to a HAL document using this profile.

Attributes

  • method string

    The HTTP method used for submitting the form

  • href string

    The target uri that the form should be submitted to

  • name string

    A string used to indentify the form.

  • title string

    A string used as title/label when showing the form in a user interface

  • type string

    The media type that must be set in the CONTENT-TYPE header when submiting the form

  • submit string

    A string used as label for the CallToAction (e.g. the submit button text)

  • fields string

    A list of field descriptors

    Nested attributes
    • fields.name string

      A string that should be used as key for the field.

    • fields.type string

      A string specifying the possible values accepted by the field.

    • fields.title string

      A string used to present this field in a UI. If not present name may be used instead.

    • fields.value string

      A prefilled value. Should be shown in a user interface (unless hidden) and sent back on submission unless changed.

    • fields.required boolean

      A boolean specifying that the field must be given a value before the form is submitted.

    • fields.hidden boolean

      A boolean specifying that the field should not be shown in a user interface.

    • fields.accepted_values array

      A list of av acceptable values. Submitting a value not present in this list SHOULD result in a client error response.
